# นำ Railway Track Engineer.AI ขึ้น EdgeOne Pages

> **ใช้ภายในหน่วยงาน — ห้ามเผยแพร่ไฟล์/คีย์สู่สาธารณะ**

EdgeOne Pages ไม่รันเซิร์ฟเวอร์ Node/Express แบบรันค้าง แต่ใช้ **Pages Functions (Serverless)** เราจึงแปลง `server.js` เดิมให้เป็นฟังก์ชัน 1 ไฟล์ ส่วนหน้าเว็บใช้ของเดิม (เรียก `/api/chat` อยู่แล้ว ไม่ต้องแก้)

---

## โครงสร้างไฟล์ (พร้อมอัปขึ้น Git / CLI)
```
railway-engineer-edgeone/
├── index.html                     ← หน้าเว็บ (static) เสิร์ฟที่ /
└── cloud-functions/
    └── api/
        └── chat.js                ← ฟังก์ชัน เสิร์ฟที่เส้นทาง /api/chat
```
- คีย์ **ไม่อยู่ในไฟล์** — ตั้งใน Environment Variables ของโปรเจกต์ EdgeOne
- รูปที่ผู้ใช้แนบจะถูกย่อในเบราว์เซอร์ก่อนส่ง (≤1568px JPEG) เพื่อให้ payload เล็ก

---

## วิธีที่ 1 — Deploy ผ่าน Git (แนะนำ)
1. นำโฟลเดอร์นี้ขึ้น repo (GitHub / GitLab / Gitee)
2. เข้าคอนโซล **EdgeOne → Pages → Create project (สร้างโปรเจกต์)** แล้วเชื่อมต่อ repo
3. ตั้งค่าโปรเจกต์
   - Framework preset: **None / Static**
   - Output directory (โฟลเดอร์ผลลัพธ์): **`/`** (รากของ repo ที่มี `index.html`)
   - ฟังก์ชันในโฟลเดอร์ `cloud-functions/` ระบบจะรับรู้และทำเส้นทางให้อัตโนมัติ
4. ไปที่ **Project settings → Environment variables** เพิ่ม
   - `ANTHROPIC_API_KEY` = **คีย์ใหม่ของท่าน** (ที่ออกหลัง revoke คีย์เดิม)
   - `MODEL` = `claude-sonnet-4-6` (หรือ `claude-opus-4-8`)
5. กด **Deploy** → เปิดโดเมนที่ระบบให้ (เช่น `xxxx.edgeone.app`) หรือผูกโดเมนหน่วยงาน
6. เปิดหน้าเว็บ → ใช้งานได้ทันที (ผู้ใช้ไม่ต้องกรอกคีย์)

## วิธีที่ 2 — ทดสอบ/ขึ้นด้วย EdgeOne CLI
1. ติดตั้ง **EdgeOne CLI** (ดูคำสั่งติดตั้งล่าสุดในเอกสาร EdgeOne Pages)
2. ทดสอบในเครื่อง: `edgeone pages dev` แล้วเปิด `http://localhost:8088`
   - ตั้งคีย์ตอนทดสอบผ่านไฟล์ `.env` ในเครื่อง หรือผ่านพารามิเตอร์ของ CLI
3. เชื่อมโปรเจกต์และดีพลอยตามคำสั่งของ CLI แล้วตั้ง Environment variables ในคอนโซลเช่นเดียวกับวิธีที่ 1

## ข้อควรระวังสำคัญ (อ่านก่อนใช้งานจริง)

1. **เลือก Region ให้เรียกถึง Anthropic ได้**
   EdgeOne เป็นแพลตฟอร์มของ Tencent และมีโหนดทั้งในจีนแผ่นดินใหญ่และต่างประเทศ — โหนดในจีนแผ่นดินใหญ่ **อาจเข้าถึง `api.anthropic.com` ไม่ได้** ให้ตั้งให้ฟังก์ชันทำงานใน **Region ต่างประเทศ (overseas)** และทดสอบว่าเรียก API สำเร็จ

2. **ความปลอดภัยของการเข้าถึง**
   เวอร์ชันนี้ยัง **ไม่มีการยืนยันตัวตนผู้ใช้** — ใครเข้าถึงหน้าเว็บได้จะเรียกใช้คีย์ผ่านฟังก์ชันได้ ควร
   - จำกัดด้วยฟีเจอร์ความปลอดภัยของ EdgeOne (เช่น WAF / IP allowlist / ควบคุมการเข้าถึง) หรือ
   - ผูกหลังระบบยืนยันตัวตนของหน่วยงาน
   - ตั้ง **Usage limit** ของคีย์ใน Anthropic Console เพื่อคุมค่าใช้จ่าย และ **หมุนคีย์เป็นระยะ**

3. **ข้อจำกัดของฟังก์ชัน**
   - คำตอบจาก Claude อาจใช้เวลาหลายวินาที และรูปทำให้ payload ใหญ่ขึ้น — หากชนลิมิตเวลา/ขนาด ให้ลดจำนวน/ขนาดรูป (ระบบย่อรูปให้แล้ว แต่ลดจำนวนรูปต่อครั้งได้)
   - Pages Functions ช่วงนี้เป็นบริการทดลองใช้ฟรีและมี rate limit อาจเปลี่ยนแปลงได้

4. **ยังไม่มีการค้นเอกสาร (RAG)**
   เวอร์ชันนี้ฝังเพอร์โซนา + กระบวนการคิดผู้เชี่ยวชาญ + วิเคราะห์รูป แต่ **ยังไม่ค้นเนื้อหาไฟล์ 23 ฉบับ** หากต้องการอ้างเลขข้อ/มาตราจากเอกสารจริง ต้องเพิ่มระบบค้นเอกสาร — ระหว่างนี้ใช้ **หน้าแชตของโปรเจกต์ใน Claude** ได้ (อ้างเอกสารได้ครบและไม่ต้องใช้คีย์)

---

## สรุป
- หน้าเว็บ = `index.html` (static, เสิร์ฟที่ `/`)
- API = `cloud-functions/api/chat.js` (เส้นทาง `/api/chat`, ใช้ `onRequestPost`)
- คีย์อยู่ใน **Environment Variables** ของ EdgeOne เท่านั้น → ผู้ใช้ไม่ต้องกรอกและมองไม่เห็น
- อย่าลืม **เลือก Region ต่างประเทศ** และ **จำกัดการเข้าถึงหน้าเว็บ**

## แก้ปัญหาที่พบบ่อย (Troubleshooting)

**อาการ: กล่องคำตอบขึ้นเป็นโค้ด HTML (มีคำว่า "Powered by pages.edgeone.ai")**
แปลว่าเรียก `/api/chat` แล้วได้ "หน้า HTML 404 ของ EdgeOne" กลับมา = ฟังก์ชันยังไม่ถูกเสิร์ฟที่เส้นทางนั้น ตรวจตามนี้
1. ฟังก์ชันต้องอยู่ที่ **`cloud-functions/api/chat.js`** จริง (ไม่ใช่โฟลเดอร์อื่น) แล้ว **Redeploy**
2. เปิด `https://<โดเมนของคุณ>/api/chat` ตรง ๆ ด้วยเมธอด GET — ถ้าเป็นหน้า 404 HTML แสดงว่าเส้นทางฟังก์ชันยังไม่ทำงาน (ควรได้ข้อความ/แมปฟังก์ชัน ไม่ใช่หน้าเว็บ)
3. ตรวจว่า EdgeOne รับรู้โฟลเดอร์ฟังก์ชันตอน build (ดู build log) — โปรเจกต์แบบ static + functions ให้ตั้ง Output directory เป็นรากที่มี `index.html`

**อาการ: ขึ้นว่า "ส่งข้อมูลกลับมาไม่ใช่ JSON" หรือ timeout**
- มักเกิดจากโหนดเข้าถึง `api.anthropic.com` ไม่ได้ → **เปลี่ยน Region ของฟังก์ชันเป็นต่างประเทศ (overseas)**
- หรือคำตอบใช้เวลานาน/รูปใหญ่เกินไป → ลดจำนวนรูป (ระบบจำกัด 3 รูปและบีบอัดให้แล้ว)

**อาการ: ขึ้นว่ายังไม่ได้ตั้งค่า ANTHROPIC_API_KEY**
- ไปที่ Project settings → Environment variables ใส่ `ANTHROPIC_API_KEY` (คีย์ใหม่) แล้ว **Redeploy** (ค่า env จะมีผลหลัง redeploy)
- ตั้งชื่อโมเดลด้วย `ANTHROPIC_MODEL` (หรือ `MODEL`) = `claude-sonnet-4-6` หรือ `claude-opus-4-8`

**ความปลอดภัยของคีย์**
- ตั้งคีย์เป็น **Secret/Environment Variable บน EdgeOne เท่านั้น** ห้ามใส่ในโค้ดหรือพิมพ์ในที่สาธารณะ
- หากคีย์เคยถูกเปิดเผย ให้ **revoke แล้วออกใหม่** ทันที และตั้ง usage limit ใน Anthropic Console
